================================================================================
HOW TO USE IN YOUR OWN COMPONENTS
================================================================================

1. Import the hook:
   import { useWebSocket } from '@/hooks/useWebSocket'
   import { useAuth } from '@/lib/auth-context'

2. Use in component:
   export function MyComponent() {
     const { token, apiUrl } = useAuth()
     
     const { isConnected, lastMessage } = useWebSocket({
       url: `${apiUrl?.replace('http', 'ws')}/ws/incidents`,
       token: token || '',
       enabled: !!token,
       onMessage: (message) => {
         if (message.type === 'incident_created') {
           console.log('New incident:', message.data)
         }
       },
     })
     
     return (
       <div>
         Status: {isConnected ? 'Live' : 'Offline'}
       </div>
     )
   }

================================================================================
TROUBLESHOOTING
================================================================================

Q: WebSocket shows "Offline" in incidents page
A: Check:
   - Token exists in localStorage
   - NEXT_PUBLIC_API_URL environment variable is set
   - Backend is running
   - CORS_ORIGINS includes your frontend domain
   - Check browser console for error messages

Q: "Missing authentication token" error
A: Make sure:
   - You're logged in (token should be in localStorage)
   - Page waited for auth context to load (useAuth() hook)
   - Token is valid (not expired)

Q: WebSocket keeps reconnecting
A: Check:
   - Backend CORS configuration
   - Network connection stability
   - Check server logs: docker logs -f [container]
   - Verify JWT token expiration time

Q: Real-time updates not working
A: Verify:
   - WebSocket connection shows "Live" in UI
   - Browser console shows "📨 WebSocket message" logs
   - Message type matches your listener (incident_created, incident_updated)
   - Incident list automatically refreshes on new incident

================================================================================
PERFORMANCE NOTES
================================================================================

- Each WebSocket connection uses minimal resources
- Connections are per-user (scoped to organization)
- Auto-cleanup on page close or logout
- Exponential backoff prevents reconnection spam
- Works in the background (no UI blocking)

Expected connection count:
- Small team (5 users): ~5 active connections
- Medium team (50 users): ~50 active connections
- Large team (500+ users): Consider Redis pub/sub or message queue
