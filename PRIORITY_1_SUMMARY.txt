API CHANGES
================================================================================

INCIDENTS ENDPOINT
GET /api/incidents
  Parameters:
    - search (string): Search title/description
    - status (string): OPEN|INVESTIGATING|RESOLVED|CLOSED
    - severity (string): CRITICAL|HIGH|MEDIUM|LOW
    - incident_type (string): data_breach|malware|...
    - skip (int): Pagination offset (default: 0)
    - limit (int): Page size (default: 50, max: 100)
  
  Example:
    GET /api/incidents?search=malware&severity=CRITICAL&skip=0&limit=10

VULNERABILITIES ENDPOINT
GET /api/vulnerabilities
  Parameters:
    - search (string): Search CVE/title/description
    - status (string): UNPATCHED|PATCHED|MITIGATED|MONITORING
    - severity (string): CRITICAL|HIGH|MEDIUM|LOW|INFO
    - skip (int): Pagination offset
    - limit (int): Page size (default: 50, max: 100)
  
  Example:
    GET /api/vulnerabilities?search=CVE-2024&severity=HIGH&skip=0&limit=10

EMAIL NOTIFICATIONS
POST /api/incidents (automatically sends emails to subscribed users)
POST /api/vulnerabilities (sends emails if severity >= HIGH)

WEBHOOKS
POST /api/integrations/webhooks (create)
GET /api/integrations/webhooks (list)
PUT /api/integrations/webhooks/{id} (update)
DELETE /api/integrations/webhooks/{id} (delete)

RATE LIMITING
All endpoints: 100 requests per minute per IP
Returns 429 on limit exceeded
