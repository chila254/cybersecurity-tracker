2. BULK ACTIONS
   Location: components/BulkActionBar.tsx
   
   Integration Steps:
   1. Add state for selected items:
      const [selectedIds, setSelectedIds] = useState<Set<string>>(new Set())
   
   2. Add checkboxes to incident list:
      <input
        type="checkbox"
        checked={selectedIds.has(incident.id)}
        onChange={(e) => {
          const newSet = new Set(selectedIds)
          if (e.target.checked) newSet.add(incident.id)
          else newSet.delete(incident.id)
          setSelectedIds(newSet)
        }}
      />
   
   3. Use BulkActionBar component:
      <BulkActionBar
        selectedCount={selectedIds.size}
        onBulkStatusChange={async (status) => {
          for (const id of selectedIds) {
            await apiClient.patch(`/incidents/${id}`, { status })
          }
        }}
        onBulkDelete={async () => {
          for (const id of selectedIds) {
            await apiClient.delete(`/incidents/${id}`)
          }
        }}
        onClose={() => setSelectedIds(new Set())}
      />
   
   Backend Endpoints Needed:

5. PDF REPORTS
   Location: backend/app/services/pdf_service.py
   
   API Endpoint to Create:
   - GET /api/reports/incidents/pdf?days=30
   - GET /api/reports/vulnerabilities/pdf?days=30
   
   Backend Implementation:
   from app.services.pdf_service import PDFReportService
   
   @router.get("/reports/incidents/pdf")
   async def get_incident_report_pdf(
       days: int = Query(30, ge=1, le=365),
       db: Session = Depends(get_db),
       current_user: dict = Depends(get_current_user)
   ):
       pdf_bytes = PDFReportService.generate_incident_report(
           db, current_user["org_id"], days
       )
       return StreamingResponse(
           io.BytesIO(pdf_bytes),
           media_type="application/pdf",
           headers={"Content-Disposition": "attachment; filename=incident_report.pdf"}
       )
   
   Frontend Usage:
   const downloadReport = async () => {
     const response = await fetch('/api/reports/incidents/pdf?days=30')
     const blob = await response.blob()
     const url = window.URL.createObjectURL(blob)
     const a = document.createElement('a')
     a.href = url
     a.download = 'incident_report.pdf'
     a.click()
   }
   
   Report Contents:
   - Executive Summary (metrics)
   - Severity Distribution
   - Recent Incidents Table
   - Recommendations
   - Professional formatting with tables

6. RESPONSIVE DESIGN UTILITIES
   Location: lib/responsive.ts
   
   Usage:
   import { responsiveClasses, isMobile, isTablet, isDesktop } from '@/lib/responsive'
   
   Tailwind Classes:
   - responsiveClasses.pageX: Responsive horizontal padding
   - responsiveClasses.headingLg: Responsive heading sizes
   - responsiveClasses.gridAuto: 1 col (mobile) → 2 (tablet) → 3 (desktop)
   - responsiveClasses.spacingMd: Responsive spacing
   
   Example Usage:
   <div className={responsiveClasses.page}>
     <h1 className={responsiveClasses.headingLg}>Title</h1>
     <div className={responsiveClasses.gridAuto}>
       {/* Items that are responsive */}
     </div>
   </div>
   
   Device Detection:
   - const mobile = isMobile() - true if screen < 768px
   - const tablet = isTablet() - true if 768px - 1024px
   - const desktop = isDesktop() - true if > 1024px
   
   Breakpoints:
   - xs: 0px (phones)
   - sm: 640px (small phones)
   - md: 768px (tablets, small laptops)
   - lg: 1024px (desktops)
   - xl: 1280px (large desktops)
   - 2xl: 1536px (extra large)

7. REDIS CACHING (Backend Enhancement)
   Location: Backend configuration
   
   Setup Redis:
   - Local: `redis-server`
   - Docker: `docker run -d -p 6379:6379 redis`
   - Environment: `REDIS_URL=redis://localhost:6379`
   
   Usage Pattern:
   from redis import Redis
   
   redis_client = Redis(host='localhost', port=6379, decode_responses=True)
   
   # Cache dashboard metrics
   cache_key = f"dashboard:metrics:{org_id}"
   cached = redis_client.get(cache_key)
   if cached:
       return json.loads(cached)
   
   metrics = calculate_metrics(org_id)
   redis_client.setex(cache_key, 300, json.dumps(metrics))  # 5 min TTL
   return metrics
   
   Benefits:
   - Reduce database queries by 70%
   - Faster dashboard loads
   - Less strain on PostgreSQL
   - Auto-expire cached data
